# Buenas practicas

Objetivos
- Comprender las mejores prácticas para la escritura de código Python.
- Aprender a documentar eficazmente el código y explicar su propósito.

Contenido
Las buenas prácticas son fundamentales para escribir código limpio, legible y mantenible. En Python, se recomienda seguir ciertas convenciones que mejoran la claridad del código y facilitan su colaboración entre desarrolladores. Una de las más importantes es el uso de la notación PEP 8, una guía de estilo para escribir códigos en Python. Entre sus recomendaciones se encuentran: usar espacios en lugar de tabulaciones para indentar, limitar la longitud de las líneas a 79 caracteres y nombrar variables con minúsculas y underscores (snake_case). Además, es crucial documentar el código de manera clara y concisa utilizando comentarios y docstrings, siguiendo la convención de PEP 257 que establece cómo se debe escribir la documentación.

Ejercicio
Implementa una función en Python que calcule el factorial de un número. Asegúrate de que tu código cumpla con las mejores prácticas aprendidas en esta lección: utiliza espacios para indentar, limita la longitud de tus líneas y nombra correctamente tus variables. Documenta tu función utilizando docstrings siguiendo PEP 257.

Resumen
- Se deben seguir convenciones como PEP 8 para mejorar la claridad del código.
- Es importante documentar el código con comentarios y docstrings según PEP 257.
