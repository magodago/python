# Buenas practicas

Objetivos
- Comprender y aplicar las mejores prácticas en la escritura de código Python.
- Familiarizarse con la importancia del estilo uniforme en proyectos de equipo.

Contenido
En esta lección, profundizaremos en las buenas prácticas para escribir código Python eficiente y legible. Una práctica crucial es seguir el estándar PEP 8, que establece convenciones sobre espaciado, indentación, nombres de variables y más. Otro aspecto importante es modularizar el código, dividiendo tareas complejas en funciones pequeñas y reutilizables. Esto no solo facilita la lectura del código sino también su mantenimiento a largo plazo.

Ejercicio
Implementa una función que calcule el factorial de un número utilizando recursión. Asegúrate de seguir las mejores prácticas aprendidas, como darle al método un nombre descriptivo y usar comentarios para explicar la lógica del código si es necesario.

Resumen
- Siguiendo PEP 8 mejora la legibilidad y mantenibilidad del código.
- Modularizar el código en funciones pequeñas y reutilizables facilita su comprensión y mantenimiento.
