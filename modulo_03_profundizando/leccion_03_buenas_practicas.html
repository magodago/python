Módulo 3: Profundizando

Buenas practicas

  • Familiarizarse con las mejores prácticas de codificación en Python para mejorar la legibilidad y mantenibilidad del código.
  • Implementar patrones de diseño y principios SOLID para resolver problemas comunes en el desarrollo de software.

Python es una lengua de programación versátil que se utiliza en una amplia gama de aplicaciones, desde desarrollo web hasta inteligencia artificial. Para aprovechar al máximo Python y garantizar que el código sea fácil de mantener y escalar, es crucial seguir ciertas buenas prácticas. Estas incluyen la utilización de nombres descriptivos para variables y funciones, la indentación correcta para mejorar la legibilidad del código, y la división clara entre diferentes bloques de código mediante comentarios o secciones.

Como ejercicio, los estudiantes deben revisar un trozo de código que no sigue estas buenas prácticas y aplicar las mejoras necesarias. Por ejemplo:

```python

def calcula_area(a,b,c):

s=(a+b+c)/2

area=(s*(s-a)*(s-b)*(s-c))**0.5

return area

print(calcula_area(3,4,5))

```

Los estudiantes deben reescribir este código siguiendo las buenas prácticas mencionadas, como el uso de nombres descriptivos para las variables y la función, así como la correcta indentación.

    Las buenas prácticas en Python son fundamentales para crear código que sea fácil de leer, mantener y escalar. Algunas de estas prácticas incluyen:

    1. **Nombres descriptivos**: Utilizar nombres claros y significativos para variables, funciones y clases.

    2. **Indentación correcta**: Python utiliza la indentación como delimitador de bloques de código, por lo que es crucial mantener una consistencia en el uso de espacios o tabulaciones.

    3. **Documentación**: Escribir documentación clara y concisa utilizando comentarios o docstrings para explicar el propósito y funcionamiento del código.

    Estas prácticas no solo mejoran la calidad del código, sino que también facilitan su mantenimiento y colaboración en equipos de desarrollo.
