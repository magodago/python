Módulo 3: Profundizando

Buenas practicas

  • Comprender y aplicar las buenas prácticas de codificación para mejorar la legibilidad y mantenibilidad del código.
  • Implementar estándares de documentación y estilo en proyectos de Python para facilitar el trabajo en equipo.

Python es una de las lenguas de programación más populares por su simplicidad y versatilidad. Para aprovechar al máximo esta potencia, resulta crucial adoptar buenas prácticas que no solo mejoren la calidad del código, sino también su mantenibilidad a largo plazo. En este módulo profundizaremos en las mejores prácticas de codificación para Python, enfocándonos en aspectos como la estructura de los archivos y directorios, el uso de comentarios y documentación, y la escritura de funciones modularizadas.

Es importante recordar que el código debe ser fácilmente comprensible no solo por su autor, sino también por otros desarrolladores. Por ejemplo, se recomienda dividir el código en módulos y clases con nombres descriptivos para facilitar la navegación y comprensión del flujo de trabajo. Además, la documentación debe ser clara y completa, incluyendo docstrings que expliquen la funcionalidad de cada función o método.

### Ejercicio

Crea un script en Python que calcule el área y el perímetro de un círculo dado su radio. Utiliza las buenas prácticas aprendidas en esta lección:

1. Define una función `calcular_area` que reciba el radio como parámetro y devuelva el área del círculo.

2. Define otra función `calcular_perimetro` que reciba el radio como parámetro y devuelva el perímetro del círculo.

3. Documenta ambas funciones con docstrings descriptivas.

4. Escribe un programa principal que solicite al usuario ingresar el radio, calcule el área y el perímetro utilizando las funciones definidas, e imprima los resultados.

### Resumen

  • La modularización de código en funciones y módulos facilita la lectura y mantenimiento del mismo.
  • La documentación detallada con docstrings es crucial para que otros desarrolladores puedan entender fácilmente el propósito y funcionamiento de cada parte del código.
